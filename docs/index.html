Everything you need

From experiment tracking to AI-powered research orchestration

📊

Run Tracking

Monitor training runs in real time with live charts, metrics, and event streams. Never lose track of an experiment again.

🤖

AI Agent

Chat with an agent that understands your project. It can launch runs, analyze results, and manage your research workflow.

🔄

Sweep Management

Define hyperparameter sweeps and let the system manage configurations, scheduling, and result aggregation.

💬

Inline Chat

Ask the agent questions while you work. Get insights, debug issues, or kick off new experiments — no context switching.

⚡

OpenCode Integration

Powered by OpenCode for agent execution and streaming. Agents run in isolated sessions with full tool access.

🖥️

tmux Background Jobs

Long-running training jobs execute in tmux sessions. Monitor, attach, or let agents manage them autonomously.

Get started in 5 minutes

Developer setup if you prefer the step-by-step approach

1

Clone the repo

git clone https://github.com/hao-ai-lab/research-agent.git && cd research-agent
2

Install (dev mode)

Sets up Node.js dependencies and Python virtual environment.

bash install.sh --dev
3

Start everything

Launches OpenCode, backend, and frontend in a tmux session.

research-agent start --project-root "$PWD"
✦

Useful commands

Check status, expose via tunnel, or stop all services.

research-agent status | tunnel | stop

Modular architecture

Four components, one tmux session, zero infrastructure

Frontend
Next.js (App Router)
Runs, sweeps, charts, events, and chat panels
Backend
FastAPI + Python
REST API for orchestration, agent routing, and job management
Agent Runtime
OpenCode
Isolated agent sessions with streaming on localhost:4096
Execution
tmux Sessions
Background job execution for long-running training runs
